User Cases

The TV Tuner API is aimed to cover the following user cases:

  • The web content is able to control the TV channel of the TV device with the TV tuner.
  • One TV can have multiple TV tuners. For example, one is for watching TV and the other is for recording.
  • TV Tuner can scan broadcasting channels, where each TV tuner can have its own scanning algorithm provided by the vendor.
  • Web content needs to know which broadcasting types are supported by the TV tuners.
  • Web content needs to know the information of Electronic Program Guide (EPG) that is being broadcasted.
  • Users can navigate through the EPG and selects a program to watch.
  • Users can watch the broadcasting program while browsing through the list of other programs that are currently broadcasted.
  • Web content needs to know if the currently displayed TV channel has bad or no signal. Can be covered by VideoElement?
  • One TV channel can have multiple video/audio tracks in the same broadcasting stream.
  • Emergency broadcasting for earthquake, tsunami... etc.
  • In the first step, only certified apps are allowed to use the TV Tuner API.
  • The DTV can keep track of the multiple TV tuners that a TV channel can belong to, so that web content can do more flexible services like: when the user is using the TV tuner A to watch and record a certain TV channel at the same time, supposing the user wants to switch to other channels to watch, the DTV should use the TV tuner B to switch channels so that it won't stop the original recording task on the TV tuner A.
  • EPG can be dynamically updated by the broadcasters. We need a way to notify the web contents to update its EPG menu from time to time.
  • It is necessary to support multiple channels that can be concurrently displayed on the TV (i.e. Picture-in-Picture).

TVTunerManager Interface

The TVTunerManager interface represents a bunch of properties and a set of operations that can be used to control the TV tuner.

readonly attribute TVChannel[] currentChannels
MUST return the TV channels that are concurrently displayed on the TV device.
Promise getAvailableTuners ()
This method makes a request to retrieve all the TV tuners that are available in the TV device. It returns a new Promise that will be used to notify the caller about the result of the operation, which is an array of TVTuner elements.
Promise getAvailableChannels ()
This method makes a request to retrieve all the TV channels that are available in a TV tuner based on the tunerId parameter. It returns a new Promise that will be used to notify the caller about the result of the operation, which is an array of TVChannel elements that belong to the TV tuner.
optional DOMString tunerId
Identifies the TV tuner that is requested to retrieve its belonging TV channels. If this parameter is absent or cannot be identified, this method will return all the TV channels that are available across TV tuners.
Promise getAvailablePrograms ()
This method makes a request to retrieve all the TV programs that are available in a TV channel based on the channelNumber parameter. It returns a new Promise that will be used to notify the caller about the result of the operation, which is an array of TVProgram elements that belong to the TV channel.
optional DOMString channelNumber
Specifies the number of the TV channel that is requested to retrieve its belonging TV programs. If this parameter is absent or cannot be identified, this method will return all the TV programs that are available across TV channels.
Promise scan ()
This method makes a request to scan all the TV channels that are available across TV tuners. It returns a new Promise that will be used to notify the caller about the result of the operation.
Promise setChannel ()
This method makes a request to set the TV channel by the options parameter. It returns a new Promise that will be used to notify the caller about the result of the operation.
SetChannelOptions options
Specifies the options for setting the TV channel.
attribute EventHandler onavailabletuneradded
Handles the onavailabletuneradded event of type AvailableTunerChangedEvent, fired when an available TV tuner is added by the TV device.
attribute EventHandler onavailabletunerremoved
Handles the onavailabletunerremoved event of type AvailableTunerChangedEvent, fired when an available TV tuner is removed by the TV device.
attribute EventHandler onavailablechanneladded
Handles the onavailablechanneladded event of type AvailableChannelChangedEvent, fired when an available TV channel is added by the TV tunner.
attribute EventHandler onavailablechannelremoved
Handles the onavailablechannelremoved event of type AvailableChannelChangedEvent, fired when an available TV channel is removed by the TV tunner.
attribute EventHandler onavailableprogramadded
Handles the onavailableprogramadded event of type AvailableProgramChangedEvent, fired when an available TV program is added in the TV channel.
attribute EventHandler onavailableprogramremoved
Handles the onavailableprogramremoved event of type AvailableProgramChangedEvent, fired when an available TV program is removed in the TV channel.
attribute EventHandler oncurrentchanneladded
Handles the oncurrentchanneladded event of type CurrentChannelChangedEvent, fired when a currently displayed TV channel is added on the TV device.
attribute EventHandler oncurrentchannelremoved
Handles the oncurrentchannelremoved event of type CurrentChannelChangedEvent, fired when a currently displayed TV channel is removed on the TV device.

Steps

The getAvailableTuners method when invoked MUST run the following steps:

  1. Let promise be a new Promise object and resolver be its associated resolver.
  2. Return promise to the caller.
  3. Make a request to the TV device to retrieve all the TV tuners that are available in the TV device.
  4. If an error occurs invoke resolver's reject algorithm with error as the value argument.
  5. When the request has been successfully completed:
    1. Let tuners be the array of the retrieved TVTuner elements.
    2. Invoke resolver's fulfill algorithm with tuners as the value argument.

The getAvailableChannels method when invoked MUST run the following steps:

  1. Let promise be a new Promise object and resolver be its associated resolver.
  2. Return promise to the caller.
  3. Make a request to the TV tuners to retrieve all the TV channels that are available in a TV tuner with the tuner ID equal to the tunerId parameter passed in the request. If the tunerId is not passed, retrieve all the TV channels that are available across TV tuners.
  4. If an error occurs invoke resolver's reject algorithm with error as the value argument.
  5. When the request has been successfully completed:
    1. Let channels be the array of the retrieved TVChannel elements.
    2. Invoke resolver's fulfill algorithm with channels as the value argument.

The getAvailablePrograms method when invoked MUST run the following steps:

  1. Let promise be a new Promise object and resolver be its associated resolver.
  2. Return promise to the caller.
  3. Make a request to the TV tuners to retrieve all the TV programs that are available in a TV channel with the channel number equal to the channelNumber parameter passed in the request. If the channelNumber is not passed, retrieve all the TV programs that are available across TV channels.
  4. If an error occurs invoke resolver's reject algorithm with error as the value argument.
  5. When the request has been successfully completed:
    1. Let programs be the array of the retrieved TVProgram elements.
    2. Invoke resolver's fulfill algorithm with programs as the value argument.

The scan method when invoked MUST run the following steps:

  1. Let promise be a new Promise object and resolver be its associated resolver.
  2. Return promise to the caller.
  3. Make a request to the TV tuners to scan and retrieve all the TV channels that are available across TV tuners.
  4. If an error occurs invoke resolver's reject algorithm with error as the value argument.
  5. When the request has been successfully completed:
    1. Invoke resolver's fulfill algorithm without assigning a value to the value argument.

The setChannel method when invoked MUST run the following steps:

  1. Let promise be a new Promise object and resolver be its associated resolver.
  2. Return promise to the caller.
  3. Make a request to the TV tuner to set the TV channel according to the setting options specified in the options parameter.
  4. If an error occurs invoke resolver's reject algorithm with error as the value argument.
  5. When the request has been successfully completed:
    1. Invoke resolver's fulfill algorithm without assigning a value to the value argument.

Event handlers

The following are the event handlers (and their corresponding event types) that MUST be supported as attributes by the TVTunerManager object.

Event handler Event name Event type Short description
onavailabletuneradded availabletuneradded AvailableTunerChangedEvent Handles the information of the available TV tuner that is added by the TV device.
onavailabletunerremoved availabletunerremoved AvailableTunerChangedEvent Handles the information of the available TV tuner that is removed by the TV device.
onavailablechanneladded availablechanneladded AvailableChannelChangedEvent Handles the information of the available TV channel that is added by the TV tuner.
onavailablechannelremoved availablechannelremoved AvailableChannelChangedEvent Handles the information of the available TV channel that is removed by the TV tuner.
onavailableprogramadded availableprogramadded AvailableProgramChangedEvent Handles the information of the available TV program that is added in the TV channel.
onavailableprogramremoved availableprogramremoved AvailableProgramChangedEvent Handles the information of the available TV program that is removed in the TV channel.
oncurrentchanneladded currentchanneladded CurrentChannelChangedEvent Handles the information of the currently displayed TV channel that is added on the TV device.
oncurrentchannelremoved currentchannelremoved CurrentChannelChangedEvent Handles the information of the currently displayed TV channel that is removed on the TV device.

TVTuner Dictionary

The TVTuner dictionary contains the information related to a TV tuner.

DOMString id
MUST return the ID of the TV tuner.
TVTunerType type
MUST return the type of the TV tuner.
MediaStream stream
MUST return a MediaStream object that is currently streamed by the TV tuner, which can be displayed by the video element.

TVChannel Dictionary

The TVChannel dictionary contains the information related to a TV channel.

TVTuner[] ownerTuners
MUST return the TV tuners which the TV channel belongs to. Note that the ownerTuners can be more than one TV tuner because different TV tuners can broadcast the same TV channel.
boolean displayed
MUST return the state whether or not the channel is currently displayed on the TV device.
TVTuner? currentTuner
MUST return the TV tuner that is currently broadcasting the TV channel. MUST return null if the TV channel is not currently displayed.
TVProgram? currentProgram
MUST return the TV program that is currently broadcasted by the TV channel. MUST return null if the TV channel is not currently displayed.
DOMString number
MUST return the number of the TV channel. E.g., "12-1", "12-2", "12-2"... etc.
TVChannelType type
MUST return the type of the TV channel.
DOMString name
MUST return the name of the TV channel. E.g., "CNN", "NHK"... etc.

TVProgram Dictionary

The TVProgram dictionary contains the information related to a TV program.

TVChannel ownerChannel
MUST return the TV channel which the TV program belongs to.
DOMString title
MUST return the title of the TV program.
unsigned long long startTime
MUST return the start time of the TV program.
unsigned long long duration
MUST return the duration of the TV program.
DOMString? description
MUST return the description of the TV program.
DOMString? language
MUST return the language of the TV program.
DOMString? rating
MUST return the rating of the TV program.

SetChannelOptions Dictionary

The SetChannelOptions dictionary contains the information for setting the TV channel.

DOMString tunerId
Identifies the TV tuner that is requested to set the TV channel.
DOMString channelNumber
Specifies the number of the TV channel that is requested to be set.

AvailableTunerChangedEvent Interface

The AvailableTunerChangedEvent interface represents the event related to the available TV tuner changed by the TV device.

readonly attribute TVTuner tuner
MUST return the available TV tuner that is changed by the TV device.

AvailableChannelChangedEvent Interface

The AvailableChannelChangedEvent interface represents the event related to the available TV channel changed by the TV tuner.

readonly attribute TVChannel channel
MUST return the available TV channel that is changed by the TV tuner.

AvailableProgramChangedEvent Interface

The AvailableProgramChangedEvent interface represents the event related to the available TV program changed in the TV channel.

readonly attribute TVProgram program
MUST return the available TV program that is changed in the TV channel.

CurrentChannelChangedEvent Interface

The CurrentChannelChangedEvent interface represents the event related to the currently displayed TV channel changed on the TV device.

readonly attribute TVChannel channel
MUST return the currently displayed TV channel that is changed on the TV device.

Enumerations

The attribute type in a TVTuner can have the following values:

dvb-t
The TV tuner works for Digital Video Broadcasting - Terrestrial.
dvb-c
The TV tuner works for Digital Video Broadcasting - Cable.
dvb-s
The TV tuner works for Digital Video Broadcasting - Satellite.

The attribute type in a TVChannel can have the following values:

tv
The TV channel is broadcasted as a regular TV type.
radio
The TV channel is broadcasted as a radio type.
